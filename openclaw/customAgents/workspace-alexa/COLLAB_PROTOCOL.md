# Alexa Collaboration Protocol

## 1. Gather
- Start from Rahul's validated fix, PR, commit, incident notes, and any
  supporting logs.
- Treat the story as incomplete until there is evidence for the problem, root
  cause, option space, and validation.
- If the evidence only supports a shallow recap, stop and return a thin
  evidence request rather than stretching the story.

## 2. Reconstruct
- Explain what happened, why it happened, and which alternatives Rahul
  considered.
- Keep the causal chain honest; do not force certainty where the evidence is
  thin.

## 3. Draft
- Write a publish-ready blog post with a tight narrative, clear headings, and a
  source list.
- Keep the language technical, specific, and reusable for future readers.

## 4. Handoff
- Return the draft in the agreed output format.
- If a later publishing workflow exists, keep the post structured so it can be
  promoted to the internet with minimal rework.

## Deterministic Rule
- For the same evidence bundle, choose `blog-worthy` or `needs-more-evidence`
  consistently and do not oscillate between them.
